# 责任编辑出版检查清单

> 三审三校流程中的检查要点

---

## 一、内容审查（一审：初审编辑）

### 1.1 政治导向与合规
- [ ] 无违反国家法律法规的内容
- [ ] 无泄露国家机密或敏感信息
- [ ] 技术内容不涉及出口管制问题
- [ ] 引用案例不涉及商业机密或隐私泄露
- [ ] 无不当政治言论或敏感表述

### 1.2 版权审查
- [ ] 所有引用数据标注来源
- [ ] 案例引用改写为原创叙述，非直接复制
- [ ] 代码示例为原创或开源许可允许
- [ ] 图片/图表有合法授权或原创
- [ ] 商标名称使用正确（如 Claude®、GitHub™ 等）

### 1.3 事实核查
- [ ] 技术版本号准确（Claude Code 发布时间等）
- [ ] GitHub star 数、项目数据为最新
- [ ] 人名、公司名拼写正确
- [ ] 日期、时间线准确
- [ ] 数据引用有权威来源

---

## 二、编校质量（二审：复审编辑）

### 2.1 语言文字
- [ ] 无错别字
- [ ] 无语法错误
- [ ] 标点符号使用规范
- [ ] 数字用法统一（阿拉伯数字 vs 中文数字）
- [ ] 量和单位符合国标（GB 3100-3102）

### 2.2 体例格式
- [ ] 章节编号连续无跳号
- [ ] 标题层级统一
- [ ] 图表编号连续（图 1-1、表 2-1 等）
- [ ] 代码块格式统一
- [ ] 注释格式统一

### 2.3 技术规范
- [ ] 术语全文统一（不混用 Skill/skill）
- [ ] 英文术语首次出现附中文解释
- [ ] 代码缩进统一（2/4 空格）
- [ ] 文件名、路径格式统一

---

## 三、装帧设计（三审：终审编辑）

### 3.1 封面与扉页
- [ ] 书名准确无误
- [ ] 作者署名正确
- [ ] 出版社名称正确
- [ ] ISBN 申请完成
- [ ] CIP 数据核字申请

### 3.2 辅文完整
- [ ] 内容简介（300-500 字）
- [ ] 作者简介
- [ ] 前言/序
- [ ] 目录（各级标题完整）
- [ ] 参考文献/索引
- [ ] 后记/致谢（可选）

### 3.3 排版质量
- [ ] 页眉页脚统一
- [ ] 页码连续
- [ ] 图表清晰可辨
- [ ] 无孤行寡行
- [ ] 章节首页排版合理

---

### 4.1 技术书籍专项检查
- [ ] 代码可运行验证

### 4.2 营销素材检查

## 常见问题速查

### 技术类图书易错点
1. **版本号陷阱**：软件版本更新快，需标注"截至 xx 年 xx 月"
2. **命令行错误**：大小写敏感、空格、引号类型
3. **路径分隔符**：Windows (\\) vs Unix (/)
4. **编码问题**：中文乱码、BOM 头
5. **超链接失效**：GitHub 仓库可能更名或删除

### 版权类易错点
1. **商标符号缺失**：首次出现应有 ® 或 ™
2. **开源协议混淆**：MIT vs Apache vs GPL
3. **图片侵权**：截图是否需授权
4. **代码引用**：开源代码需标注来源和协议
