# 3.1 Skill Creator - "能生娃的Skill"

## 什么是Skill Creator

Skill Creator是一个"元技能"（Meta Skill）——它的作用不是直接帮你完成某项工作，而是帮你创建其他Skill。如果说普通的Skill是"数字员工"，那么Skill Creator就是"员工培训系统"。

这个概念的巧妙之处在于：一旦掌握了Skill Creator，你就拥有了无限扩展能力。需要处理PDF？用Skill Creator创建一个。需要分析数据？用Skill Creator创建一个。你不需要懂得复杂的编程，只需要描述清楚需求，Skill Creator就能帮你生成完整的Skill结构。

## 为什么需要Skill Creator

**降低创建门槛**：
- 不需要记忆YAML Frontmatter的语法细节
- 不需要了解Markdown正文的最佳实践
- 不需要从头构思目录结构
- 通过对话式交互完成Skill设计

**确保质量一致性**：
- 自动遵循官方推荐的结构规范
- 内置常见模式的模板
- 自动验证生成的Skill语法正确性

**加速迭代过程**：
- 从想法到可用Skill只需几分钟
- 支持基于现有Skill的改造
- 自动处理脚本和引用文件的创建

## Skill Creator的工作原理

Skill Creator通过引导式对话收集需求，然后自动生成完整的Skill文件结构：

**第一步：需求收集**
- 询问Skill的名称和用途
- 了解触发条件和执行场景
- 确认需要的工具和权限

**第二步：结构设计**
- 生成合适的YAML Frontmatter
- 设计Markdown正文的三层结构
- 规划scripts和references目录

**第三步：代码生成**
- 编写示例脚本
- 创建参考文档模板
- 生成配置文件

**第四步：验证输出**
- 检查语法正确性
- 验证结构完整性
- 提供测试建议

## 如何使用Skill Creator

### 安装方法

```bash
# 通过Claude安装
帮我安装 skill，地址是 https://github.com/anthropics/skills/tree/main/skills/skill-creator
```

### 使用流程

**示例：创建一个周报生成Skill**

1. **启动Skill Creator**
   ```
   用户：使用 skill-creator 帮我做一个分析 Excel 的 skill
   ```

2. **回答引导问题**

   Skill Creator会依次询问：
   - "这个Skill的名称是什么？" → `weekly-report-generator`
   - "它应该在什么情况下被触发？" → "当用户需要生成工作周报时"
   - "它需要访问哪些工具？" → "Read, Bash, Git"
   - "它的主要功能是什么？" → "基于Git提交记录生成周报"

3. **自动生成结构**

   Skill Creator会生成如下结构：
   ```
   weekly-report-generator/
   ├── SKILL.md
   ├── scripts/
   │   └── parse_git_commits.py
   └── references/
       └── report_template.md
   ```

4. **查看生成的SKILL.md**

   ```yaml
   ---
   name: weekly-report-generator
   description: When user needs to generate a weekly work report based on Git commits
   user-invocable: true
   allowed-tools:
     - Read
     - Bash
     - Grep
   ---

   ## 触发条件

   当用户需要生成周报时激活：
   - "帮我生成本周周报"
   - "总结一下这周做了什么"
   - "看看这周提交了哪些代码"

   ## 执行步骤

   1. **收集Git提交记录**
      ```bash
      git log --since="last monday" --pretty=format:"%h %s"
      ```

   2. **解析提交信息**
      运行 scripts/parse_git_commits.py 解析提交记录

   3. **生成周报内容**
      基于模板生成结构化周报

   4. **输出结果**
      保存到 weekly-report.md
   ```

5. **微调细节**

   你可以要求Skill Creator修改：
   - "把输出格式改成Markdown表格"
   - "添加统计代码行数的功能"
   - "支持导出为PDF"

## 实战案例：创建数据可视化Skill

**需求描述**：
"我需要创建一个Skill，能够自动分析CSV文件并生成数据可视化图表。"

**Skill Creator的对话过程**：

```
Claude: 我来帮你创建这个数据可视化Skill。首先，这个Skill叫什么名字？

用户: csv-visualizer

Claude: 好的。这个Skill应该在什么情况下被触发？

用户: 当用户需要分析CSV文件并生成图表时

Claude: 它需要哪些工具权限？比如是否需要运行Python脚本？

用户: 需要Read读取文件，Bash运行Python脚本，Write生成图表文件

Claude: 你希望生成什么格式的图表？PNG图片还是HTML交互式？

用户: 先支持PNG吧，要有柱状图和折线图

Claude: 好的，我现在为你生成这个Skill的完整结构。
```

**生成的文件结构**：

```
csv-visualizer/
├── SKILL.md
├── scripts/
│   ├── analyze_csv.py      # CSV数据分析脚本
│   └── generate_chart.py   # 图表生成脚本
├── references/
│   └── chart_styles.md     # 图表样式参考
└── requirements.txt        # Python依赖
```

**生成的analyze_csv.py脚本**：

```python
#!/usr/bin/env python3
"""CSV数据分析和图表生成"""
import pandas as pd
import matplotlib.pyplot as plt
import sys

def analyze_csv(file_path):
    """分析CSV文件并生成统计信息"""
    df = pd.read_csv(file_path)

    stats = {
        'rows': len(df),
        'columns': list(df.columns),
        'numeric_columns': df.select_dtypes(include=['number']).columns.tolist(),
        'summary': df.describe().to_dict()
    }
    return stats

def generate_chart(file_path, chart_type='bar', output_path='chart.png'):
    """生成数据图表"""
    df = pd.read_csv(file_path)

    plt.figure(figsize=(10, 6))

    if chart_type == 'bar':
        df.plot(kind='bar')
    elif chart_type == 'line':
        df.plot(kind='line')

    plt.title('Data Visualization')
    plt.xlabel('Index')
    plt.ylabel('Value')
    plt.tight_layout()
    plt.savefig(output_path)

    return output_path

if __name__ == '__main__':
    if len(sys.argv) < 2:
        print("Usage: analyze_csv.py <csv_file>")
        sys.exit(1)

    file_path = sys.argv[1]
    stats = analyze_csv(file_path)
    print(f"Analysis complete: {stats['rows']} rows, {len(stats['columns'])} columns")
```

## Skill Creator的高级用法

### 基于现有Skill改造

你可以让Skill Creator分析现有的Skill，然后基于它创建新版本：

```
用户: 基于现有的code-reviewer skill，创建一个专门用于Python代码审查的skill

Claude: 我来分析code-reviewer skill的结构，然后为你创建python-code-reviewer。
...
```

### 批量创建Skill套件

Skill Creator还支持一次性创建多个相关的Skill：

```
用户: 帮我创建一套数据分析工具，包含数据清洗、数据可视化、数据导出三个skill

Claude: 好的，我将为你创建数据分析工具套件：
1. data-cleaner: 处理缺失值、异常值、格式转换
2. data-visualizer: 生成图表和报告
3. data-exporter: 导出为不同格式（Excel, JSON, SQL）
...
```

## 最佳实践

### 1. 明确需求边界

在使用Skill Creator之前，先想清楚：
- 这个Skill的核心功能是什么？
- 它不应该做什么？（避免功能蔓延）
- 触发条件是否清晰明确？

### 2. 迭代优化

第一版Skill往往不完美，建议：
- 先创建最小可用版本
- 实际使用后收集问题
- 用Skill Creator持续改进

### 3. 积累个人模板

对于常用的Skill模式，可以：
- 保存Skill Creator生成的优质模板
- 建立自己的Skill模板库
- 下次直接基于模板修改

Skill Creator是Agent Skills生态的"加速器"，它让创建Skill变得像对话一样简单：

**核心价值**：
- 零代码创建专业Skill
- 降低技能扩展门槛
- 确保结构规范统一

**使用要点**：
- 清晰描述需求
- 回答引导问题
- 迭代优化细节

**进阶玩法**：
- 基于现有Skill改造
- 批量创建工具套件
- 建立个人模板库

掌握Skill Creator，你就拥有了无限扩展AI能力的可能。不再受限于现有的Skill库，任何重复性工作都可以封装成自己的专属Skill。
